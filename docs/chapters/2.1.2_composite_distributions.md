### Composite distributions 
This section contains composite distributions in the sense that they refer to other distribution which they combine or modify in some way. 


#### Mixture distribution {#sec:mixture-distribution label="sec:mixture-distribution"}

The mixture distribution, sometimes called addition of distributions, is a (weighted) sum of distributions $f_i(\vec{x})$, depending on the same variable(s) $\vec{x}$: 

$$
\begin{aligned} \text{MixturePdf}(x) = \frac{1}{\ensuremath{\mathcal{M}}}\sum_{i=1}^{n} c_i \cdot f_i(\vec{x}) , \end{aligned} 
$$



where the $c_i$ are coefficients and $\vec{x}$ is the vector of variables. 

- `name`: custom unique string 
- `type`: `mixture_dist` 
- `x`: name of the variable $x$ ([optional]{.smallcaps}, since the     variable is fully defined by the summands) 
- `summands`: array of names referencing distributions 
- `coefficients`: array of names of coefficients $c_i$ or numbers to     be added 
- `extended`: boolean denoting whether this is an extended     distribution ([optional]{.smallcaps}, as it can be inferred from the     lengths of the lists for `summands` and `coefficients`) 
This distribution can be treated as extended or as non-extended. 
- If the number of coefficients equals the number of distributions and `extended` is `false`, then the sum of the coefficient must be     (approximately) one. 
- If the number of coefficients equals the number of distributions and `extended` is `true`, then the sum of the coefficients will be used     as the Poisson rate parameter and the mixture distribution itself     will use the coefficients normalized by their sum. 
- If the number of coefficients is one less than the number of     distributions, then `extended` must be `false` and $c_n$ is computed     from the other coefficients as 



$$
\begin{aligned}     c_n &= 1 - \sum_{i=1}^{n-1}c_i .     \end{aligned} 
$$


#### Mixture model

The mixture model is a weighted sum of resolution models or model PDFs:

$$
\begin{aligned}
\text{MixtureModel}(x) = \sum_{i=1}^{n} c_i\,m_i(x).
\end{aligned}
$$

If one coefficient fewer than the number of summands is provided, the final coefficient is computed as

$$
\begin{aligned}
c_n = 1 - \sum_{i=1}^{n-1} c_i .
\end{aligned}
$$

If all coefficients are given, their sum is used as the expected number of events in extended use cases.

-   `name`: custom unique string
-   `type`: `mixture_model`
-   `summands`: array of names referencing model distributions
-   `coefficients`: array of names of coefficients $c_i$ or numbers
-   `extended`: boolean denoting whether this is an extended model  


#### Indexed distribution

The indexed distribution combines severeal sub-distributions selected by the value of a discrete/categorical variabel `index`. `index` [may]{.smallcaps} refer either to a categorical axis of the data this distribution is paired with, or to a parameter with a categorical domain. Which of the two determines how the distribution is evaluated.

If `index` refers to a data axis (`exclusive: false`), evaluation is dispatched per entry:

$$
\begin{aligned}
\text{IndexedPdf}(x) &= \text{PDF}(m_{c(x)}(\theta), x)
\end{aligned}
$$

where $c(x)$ denotes the value of `index` for the variate $x$, and $m_{c(x)}$ is the sub-distribution assigned to that value/category. In the context of a likelihood, each entry of a paired data set is thus evaluated under the sub-distribution of its own category only: every member contributes terms, but only for the entries belonging to it. This is the typical construction for simultaneous fits of several measurement regions (or channels). In this scenario `index` itself is an observable.  

If `index` instead refers to a parameter with a categorical domain (`exclusive: true`), only a single member is evaluated at a time, chosen by the current value $c$ of that parameter:

$$
\begin{aligned}
\text{IndexedPdf}(x) &= \text{PDF}(m_{c}(\theta), x)
\end{aligned}
$$

Here `index` itself can be a free parameter of the fit, letting maximizing the likelihood determine the optimal choice of candidate distribution for a given data. 

A indexed distribution is a struct defined by the following components:


- `name`: custom string
- `type`: `indexed_dist`
- `index`: strict describing the categorical variable that select the member for evaluation:
    - `name`: name of the categorical variable. [must]{.smallcaps}, as a subset, refer to a categorical axis of the data this distribution is paired, or to a parameter with a categorical domain
    - `categories`: array of unique strings enumerating the category values used by this distribution
- `members`: array of structs, each assigning one category value to the distribution used for it:
    - `category`: string, [must]{.smallcaps} match to one of `index.categories`
    - `dist`: name of the sub-distribution used for this value of `category`
- `exclusive`: ([optional]{.smallcaps}) boolean, defaults to `false`. If `true`, `name` refers to a parameter, and only the single member matching that parameter's current value is evaluated. If `false` (default), `name` refers to a categorical axis of the paired data, and every member contributes jointly, dispatched per data entry.
- `correction`: ([optional]{.smallcaps}) number used as additional correction to the $-2\log{\mathcal{L}}$ contribution when this member is the active choice. Defaults to $0.5*n_p$ (`correction: 0.5`) with the number of parameter $n_p$ of the active member distribution (free model parameters + observables). Only meaningful when `exclusive` is `true`, can be used to penalize the different model complexity between the candidate member distributions. 

The distributions referenced by `members` [may]{.smallcaps} each have different variate names, since only one member's distribution is evaluated for any given category value. `members` [need not]{.smallcaps} cover every entry of `index.categories`; categories without a matching entry are undefined for this distribution. The 

```json title="Example: Indexed distribution, joint"
{
    "name":"combined_pdf",
    "type":"indexed_dist",
    "index":{"name":"channel", "categories":["SR","CR"]},
    "members":[
        {"category":"SR", "dist":"sr_pdf"},
        {"category":"CR", "dist":"cr_pdf"}
    ]
}
```

```json title="Example: Indexed distribution, exclusive"
{
    "name":"bkg_choice_pdf",
    "type":"indexed_dist",
    "index":{
        "name":"bkg_function",
        "categories":["poly1","poly2","exp1"],
    },
    "exclusive":true,
    "members":[
        {"category":"poly1", "dist":"bkg_poly1_pdf"},
        {"category":"poly2", "dist":"bkg_poly2_pdf"},
        {"category":"exp1", "dist":"bkg_exp1_pdf"}
    ],
    "correction":1.0,
}
```  


#### Product distribution {#sec:product-distribution label="sec:product-distribution"} 

The product of independent distributions $f_i$ is defined as 



$$
\begin{aligned} \text{ProductPdf}(x) &= \frac{1}{\ensuremath{\mathcal{M}}}\prod_i^n f(x). \end{aligned} 
$$



-   `name`: custom string 
-   `type`: `product_dist` 
-   `x`: name of the variable $x$ ([optional]{.smallcaps}, since the     variable is fully defined by the factors) 
-   `factors`: array of names referencing distributions 

#### Efficiency product distribution

The efficiency product distribution multiplies a normalized PDF with an efficiency function and normalizes the product:

$$
\begin{aligned}
\text{EfficiencyProductPdf}(x) =
\frac{1}{\ensuremath{\mathcal{M}}}\, f(x)\,\epsilon(x).
\end{aligned}
$$

The efficiency function is expected to be non-negative over the domain.

-   `name`: custom unique string
-   `type`: `efficiency_product_pdf_dist`
-   `pdf`: name of the input distribution $f$
-   `eff`: name of the efficiency function $\epsilon$

#### Density Function distribution 

A distribution that is specified via a non-normalized density function $f(x)$. Formally, it corresponds to the probability measure that has the density $f(x)$ in respect to the Lebesgue measure. 

The density function is normalized automatically by $\ensuremath{\mathcal{M}}= \int f(x) dx$, so the PDF of the distribution is 

$$
\begin{aligned} \text{DensityFunctionPdf}(f, x) &= \frac{f(x)}{\ensuremath{\mathcal{M}}} \end{aligned} 
$$



The distribution can be specified either via the non-normalized density function $f(x)$ or via the non-normalized log-density function $\log(f(x))$: 

-   `density_function_dist`: Specified via the PDF $f(x)$ 
    -   `name`: custom string 
    -   `type`: `density_function_dist` 
    -   `function`: The density function $f$ 
-   `log_density_function_dist`: Specified via the PDF $f(x)$ 
    -   `name`: custom string 
    -   `type`: `log_density_function_dist` 
    -   `function`: The density function $f$ 



#### DiagonalGauss distribution {#sec:diag-gaussian-distribution label="sec:diag-gaussian-distribution}


The DiagonalGauss distribution is a product of independent, one-dimensional Gaussian distributions, compactly indexed against a categorical axis of paired data:

$$
\begin{aligned}
\text{DiagonalGaussPdf}(\vec{d}) &= \prod_{i=1}^{n} \text{GaussianPdf}(d_i; \mu_i, \sigma)
\end{aligned}
$$

where $d_i$ is read from the column of the paired data set whose `index` category matches the corresponding entry of `member`. It is sematically equivalent to a [Product distribution](#sec:produc-distribution){referecen-type="ref" reference="sec:product-distribution"} of $n$ one-dimensional Gaussians, but declared flat and explicit such that engines [may]{.smallcaps} use a dedicated, vectorized implementation.  

A DiagonalGauss distribution is a struct defined by the following components:  

- `name`: custom string
- `type`: `diagonal_gauss_dist`
- `index`: struct describing the categorical variable that labels each auxiliary measurement of $d$
    - `name`: name of the categorical variable. [must]{.smallcaps} refer to a categorical axis of the data this distribution is paired with
    - `categories`: array of unique strings enmerating the category values used by this distribution
- `d_axis`: name of the continuous axis of the paired data holding $d_i$, read from the column matching the category of `index`
- `sigma`: ([optional]{.smallcaps}) number giving $\sigma$, shared by every member. Defaults to `1.0`
- `members`: array of structs, each assigning one categorical value to the parameter it maps to $mu_i$
    - `category`: string, [must]{.smallcaps} match one of `index.categories`
    - `mu`: name of the mean $\mu_i$, can a parameter name or function.

```json title="Example: DiagonalGauss distribution paired with auxiliary-measurement data"
{
    "distributions":[
        ...
            {
                "name":"gauss_constraints",
                "type":"diagonal_gauss_dist",
                "index":{"name":"np", "categories":["np_1_in","np_2_in"]},
                "mean_axis":"nom_np",
                "members":[
                    {"category":"np_1_in","x":"np_1"},
                    {"category":"np_2_in","x":"np_2"},
                ]
            }
        ...
    ],
    "data":[
        ...
        {
            "name":"nom_np",
            "type":"unbinned",
            "entries":[[0,0], [1,0]],
            "axes":[
                {"name":"np", "categories":["np_1_in","np_2_in"]},
                {"name":"gaussian_constraint_mean","min":-5, "max":5}
            ]
        }
        ...
    ]
}
```
#### DiagonalPoisson distribution {#sec:diag-poisson-distribution label="sec:diag-poisson-distribution}

The DiagonalPoisson distribution is the Poisson analogue of [DiagonalGaussian distribution](#sec:diag-gaussian-distribution){reference-type="ref" reference="sec:diag-gaussian-distribution}:

$$
\begin{aligned}
\text{DiagonalPoissonPdf}(\vec{d}) &= \prod_{i=1}^{n} \text{PoissonPdf}(d_i;\lambda_i)
\end{aligned}
$$
where $d_i$, the fixed observed auxiliary count, is read from the row of the paired data set whose `index` category matches the corresponding entry of `members`.  

- `name`: custom string
- `type`: `diagonal_poisson_dist`
- `index`: struct describing the categorical variable that labels each entry:
    - `name`: name of the categorical variable. [must]{.smallcaps} refer to a categorical axis of the data this distribution is paired with
    - `categories`: array of unique strings enumerating the category values used by this distribution
- `d_axis`: name of the continuous axis of the paired data holding $d_i$, read from the column matching the category of `index`
- `members`: array of structs, each assigning one category value to the parameter it maps to $\lambda_i$:
    - `category`: string, [must]{.smallcaps} match on of `index.categories`
    - `lambda`: name of the rate $\lambda_i$, can a parameter name or function.

```json title="Example: DiagonalPoisson distribution"
{
    "name":"poisson_constraint",
    "type":"diagonal_poisson_dist",
    "index":{"name":"bin", "categories":["bin_1_in","bin_2_in"]},
    "d_axis":"nom_bin",
    "members":[
        {"category":"bin_1_in","lambda":"rate_1"},
        {"category":"bin_2_in","lambda":"rate_2"}
    ]

}
```


#### Poisson point process {#dist:poisson-point-process label="dist:poisson-point-process"} 

A Poisson point process distribution is understood here as the distribution of the outcomes of a (typically inhomogeneous) Poisson point process. In particle physics, such a distribution is often called an extended distribution [@barlow-extended-ml].
In HS<sup>3</sup>, a Poisson point process distribution is specified using a global-rate parameter $\lambda$ and an underlying distribution $m$, either explicitly or implicitly (see below). 
Random values are drawn from the Poisson point process distribution by drawing a random number $n$ from a Poisson distribution of the global-rate $\lambda$, and then drawing $n$ random values from the underlying distribution $m$. The resulting random values are vectors $x$ of length $n$, so their length varies. 
The PDF the Poisson point process distribution at an outcome $x$ (a vector of length $n$) is 



$$
\begin{aligned} \text{PoissonPointProcessPdf}(\lambda, m, x) &= \text{PoissonPdf}(n, \lambda) \cdot \prod_i^n \text{PDF}(m, x_i). \end{aligned} 
$$



In particle physics, the function $\text{PoissonPointProcessPdf}(\lambda, m(\theta), x)$, for a fixed observation $x$ and varying parameters $\lambda$ and $\theta$, is often called an extended likelihood. 
A Poisson point process distribution can be specified in HS<sup>3</sup> in two ways: 

-   `rate_extended_dist`: The global-rate parameter $\lambda$     and underlying distribution $m$ are specified explicitly: 
    -   `name`: custom string 
    -   `type`: `rate_extended_dist` 
    -   `rate`: The global rate $\lambda$ 
    -   `dist`: The underlying distribution $m$ 

The name of the variable $x$ is taken from the underlying     distribution. The underlying distribution [must]{.smallcaps} not be     referred to from other components of the statistical model. 

-   `rate_density_dist`: Specified via a non-normalized rate-density function $f$. Both the global-rate parameter and the underlying distribution are implicit: $\lambda = \int f(y) d y$ and $m = \text{DensityFunctionPdf}(f)$. More formally, the distribution corresponds to the inhomogeneous Poisson point process that is defined by a non-normalized rate measure which has density $f$ in respect to the Lebesgue measure. 
    -   `name`: custom string 
    -   `type`: `rate_density_dist` 
    -   `x`: name of the variable $x$ 
    -   `density`: The rate-density function $f$ 

#### Bin-count distribution {#dist:bin-counts label="dist:bin-counts"} 

This is a binned version of the Poisson point process distribution 
([Poisson point process](#dist:poisson-point-process){reference-type="ref" reference="dist:poisson-point-process"}). It is the distribution of the bin counts that result from histogramming the outcomes of a Poisson point process distribution using a given binning scheme (see 
[Binned Data](#sec:binned-data){reference-type="ref" reference="sec:binned-data"}). 
Like the Poisson point process distribution, a Bin-counts distribution can either be specified via a global-rate parameter $\lambda$ and an underlying distribution $m$, or via a rate-density function $f$ (in which case $\lambda$ and $m$ are implicit). In addition, the binning scheme also has to be specified in either case, unless it can be inferred (see below). 
For $k$ bins, this type of distribution corresponds to a product of $k$ Poisson distributions with rates 



$$
\begin{aligned} \nu_i &= \lambda \cdot \int_{\text{bin}_i} \text{PDF}(y, m) dy \quad \text{equivalent to}\\ \nu_i &= \int_{\text{bin}_i} f(y) dy \end{aligned} 
$$



The PDF of the bin-counts distribution at an outcome $x$ (a vector of length $k$, same as the number of bins) is 



$$
\begin{aligned} \text{BinCountsPdf}(x) &= \prod_i^k \text{PoissonPdf}(x_i, \nu_i) \end{aligned} 
$$



-    `bincounts_extended_dist`: The global-rate parameter $\lambda$ and underlying distribution $m$ are specified explicitly: 
     -   `name`: custom string 
     -   `type`: `bincounts_extended_dist` 
     -   `rate`: The global rate $\lambda$ 
     -   `dist`: The underlying distribution $m$ 
     -   `axes`: a definition of the binning to be used, following the in [Binned Data](#sec:binned-data){reference-type="ref" reference="sec:binned-data"}. [optional]{.smallcaps} if `dist` is a binned distribution, in which case the same binning is used by default.  
	
The name of the variable $x$ is taken from the underlying distribution. The underlying distribution [must]{.smallcaps} not be referred to from other components of the statistical model. 
`bincounts_density_dist`: Specified via a non-normalized rate-density function $f$. Both the global-rate parameter and the underlying distribution are implicit: $\lambda = \int f(y) dy$ and $m = \text{DensityFunctionPdf}(f)$. 

More formally, the distribution corresponds to the inhomogeneous Poisson point process that is defined by a non-normalized rate measure which has density $f$ in respect to the Lebesgue measure. 

-   `name`: custom string 
    -   `type`: `bincounts_density_dist` 
    -   `x`: name of the variable $x$ 
    -   `density`: The rate-density function $f$ 
    -   `axes`: a definition of the binning to be used, following the definitions in [Binned Data](#sec:binned-data){reference-type="ref" reference="sec:binned-data"}.
